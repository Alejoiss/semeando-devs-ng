# Requirements

## Overview
The landing page needs a complete revamp to better communicate the platform's value proposition, learning methodology, and gamification mechanics. The current layout is confusing and does not effectively highlight the core features: diverse courses (HTML, CSS, JS, and more to come), the three lesson types (Normal, Review, Challenge), the AI-driven feedback, and the gamification system (XP, Seeds, Rankings, Achievements). The new landing page should provide a clear, direct, and attractive presentation of these elements and include strong calls to action to guide new users to start their journey.

## Requirements

### REQ-1: Presentation of Courses and Trilha

**User Story:** As a prospective user, I want to see the available courses and upcoming ones, so that I understand what I can learn on the platform.

#### Acceptance Criteria
1.1 THE application SHALL display the currently available courses.
1.2 THE application SHALL display indicators for planned future courses.

### REQ-2: Presentation of Learning Methodology

**User Story:** As a prospective user, I want to understand the types of lessons and the learning process, so that I know how the platform works.

#### Acceptance Criteria
2.1 THE application SHALL display information about the three lesson types: Normal, Review, and Challenge.
2.2 THE application SHALL describe the AI mentor review process for Challenge lessons.

### REQ-3: Presentation of Gamification Mechanics

**User Story:** As a prospective user, I want to learn about the XP, Seeds, Rankings, and Achievements, so that I am motivated to participate and progress.

#### Acceptance Criteria
3.1 THE application SHALL display an explanation of the XP and Seeds economy.
3.2 THE application SHALL display an overview of the ranking system.
3.3 THE application SHALL display an overview of the achievements system.

### REQ-4: Call to Action and Onboarding

**User Story:** As a prospective user, I want clear instructions on how to start, so that I can easily begin my learning journey.

#### Acceptance Criteria
4.1 THE application SHALL display prominent call-to-action elements for registration and starting courses.
